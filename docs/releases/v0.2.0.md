# v0.2.0 — Bespoke

The make-or-break milestone: make every generated site look genuinely
custom-designed, never templated — and enforce it with a real quality gate.

## Highlights

- **Per-run token engine.** Every run derives a unique design-token set —
  HSL-generated palette, modular type scale, corner radius, spacing, motion —
  with a short fingerprint. Two runs of the *same* idea produce visibly
  different sites; palettes deliberately steer clear of generic indigo/blue.
- **Design Critic gate.** Each candidate is scored 0–100 across four real
  dimensions — WCAG contrast, bespokeness, hierarchy, and AI-tell copy
  heuristics — with a hard veto on sub-AA contrast.
- **Self-revision loop.** Failing candidates are automatically revised up to two
  rounds (fixing contrast, pushing the palette away from generic hues) until
  they clear the gate.
- **UI/UX agent + generated logo.** A new agent runs the token engine and emits
  a seeded SVG logomark colored from the palette.
- **Token-aware renderer.** Radius, type scale, hero/feature layout variants,
  and the logo all flow from tokens, so two runs differ in shape, not just color.
- **Cockpit insight.** The UI now shows the design tokens, the fingerprint, and
  the Critic scorecard with per-dimension sub-scores and revision history.

## Under the hood

The pipeline grew from five to seven agents
(`CEO → PM → Brand → UI/UX → Copy → Frontend → Design Critic`), runs now carry a
per-run seed, and the brand palette moved into the design-token model.

## Upgrade notes

No migration needed — generation is stateless per run. Existing runs from
v0.1.0 still render via the preview route.
